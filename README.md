# Soldo — Admin Panel

Панель администратора для управления событиями, бронированиями и настройками виджета.

Доступна напрямую по URL без авторизации — защита на уровне сети.

## Стек

| Пакет | Версия | Роль |
|-------|--------|------|
| React | 19 | UI-фреймворк |
| TypeScript | 5.9 | Типизация |
| Vite | 7 | Сборщик и dev-сервер |
| React Bootstrap + Bootstrap | 2.10 / 5.3 | UI-компоненты |
| React Router | 7 | Клиентская маршрутизация |
| Axios | 1.13 | HTTP-клиент |
| Recharts | 3.7 | Графики на дашборде |
| React Icons | 5.5 | Иконки (Bootstrap Icons) |

## Быстрый старт

```bash
npm install
npm run dev
```

Dev-сервер поднимается на `http://localhost:3000`. Запросы на `/api/**` проксируются на `http://localhost:8080` (настроено в `vite.config.ts` — backend должен быть запущен).

```bash
npm run build    # production-сборка в dist/
npm run preview  # предпросмотр production-сборки
npm run lint     # ESLint
```

## Маршруты

| Путь | Компонент | Описание |
|------|-----------|----------|
| `/admin` | `Dashboard` | Дашборд: метрики, ближайшие события, выручка |
| `/admin/events` | `Events` | Список событий с фильтрами и поиском |
| `/admin/events/new` | `EventForm` | Создание события |
| `/admin/events/:id` | `EventDetail` | Детали события, список бронирований, документы |
| `/admin/events/:id/edit` | `EventForm` | Редактирование события |
| `/admin/bookings` | `Bookings` | Все события с раскрывающимися списками бронирований |
| `/admin/widget` | `WidgetSettings` | Конфигурация встраиваемого виджета |

## Страница бронирований

`Bookings.tsx` отображает аккордеон: каждый элемент — событие с числом бронирований и свободных мест. При раскрытии подгружается список бронирований для этого события. Доступные действия на каждом бронировании: отмена и отметка оплаты.

## Страница деталей события

`EventDetail.tsx` показывает полную информацию о событии, сводку мест и таблицу бронирований. Для каждого бронирования выводятся: имя участника, контакты, комментарий (`notes`), статус оплаты, дата создания. Можно отменить бронирование, отметить оплату, просмотреть документы.

## Типы (`types/index.ts`)

| Интерфейс | Описание |
|-----------|----------|
| `Event` | Событие (название, даты, цена, вместимость, статус) |
| `EventFormData` | Форма создания/редактирования события |
| `Booking` | Бронирование (гостевые данные, оплата, статус, комментарий) |
| `AdminBookingRequest` | Запрос на создание бронирования администратором |
| `BookingSummary` | Счётчики мест и бронирований для события |
| `DocumentTemplate` | Шаблон документа |
| `BookingDocument` | Документ, привязанный к бронированию |
| `WidgetConfig` / `WidgetConfigUpdateRequest` | Конфигурация виджета |

## API-клиент (`api/client.ts`)

- Базовый URL: `/api` (проксируется через Vite в dev, через Caddy в production)
- Без авторизации — backend принимает все запросы

## Docker

```bash
docker build -t soldo-admin .
docker run -p 3000:80 soldo-admin
```

`Dockerfile` — multi-stage сборка: `node:20` (Vite) → `nginx:1.27` (статика). Nginx настроен с SPA fallback (`try_files $uri /index.html`).
