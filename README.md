# Soldo — Панель администратора

React-приложение для управления событиями, бронированиями, документами и настройками виджета.

Доступна без авторизации — защита реализована на уровне сети (Caddy, firewall).

## Стек

| Библиотека | Версия | Роль |
|-----------|--------|------|
| React | 19 | UI-фреймворк |
| TypeScript | 5.9 | Типизация |
| Vite | 7 | Сборщик, dev-сервер с HMR |
| React Bootstrap + Bootstrap | 2.10 / 5.3 | UI-компоненты |
| React Router | 7 | Клиентская маршрутизация |
| Axios | 1.13 | HTTP-клиент |
| Recharts | 3.7 | Графики на дашборде |
| React Icons | 5.5 | Иконки (Bootstrap Icons) |

## Быстрый старт

```bash
npm install
npm run dev        # dev-сервер на http://localhost:3000
npm run build      # production-сборка в dist/
npm run preview    # предпросмотр production-сборки
npm run lint       # ESLint
```

Запросы на `/api/**` проксируются на `http://localhost:8080` — backend должен быть запущен (настроено в `vite.config.ts`).

## Маршруты

| Путь | Компонент | Описание |
|------|-----------|----------|
| `/admin` | `Dashboard` | Дашборд: метрики, ближайшие события, выручка |
| `/admin/events` | `Events` | Список событий с фильтрами и поиском |
| `/admin/events/new` | `EventForm` | Создание события |
| `/admin/events/:id` | `EventDetail` | Детали события, бронирования, документы |
| `/admin/events/:id/edit` | `EventForm` | Редактирование события |
| `/admin/bookings` | `Bookings` | Все события с раскрывающимися списками бронирований |
| `/admin/widget` | `WidgetSettings` | Конфигурация виджета |
| `/admin/settings` | `AppSettings` | Настройки приложения и SMTP |

## Ключевые страницы

### Dashboard (`/admin`)

Показывает: количество событий по статусам, активные бронирования, выручку за текущий месяц (график + сумма), ближайшие 5 событий.

### Events (`/admin/events`)

Таблица событий с фильтрацией по статусу (DRAFT / PUBLISHED), поиском по названию. Быстрые действия: публикация/снятие, редактирование, удаление.

### EventDetail (`/admin/events/:id`)

- Полная информация о событии и сводка мест (свободно / всего)
- Таблица бронирований: имя участника, контакты, комментарий, статус оплаты, дата создания
- Для каждого бронирования: удаление (с подтверждением), отметка оплаты
- Вкладка документов: список шаблонов, привязанных к событию, и документы каждого бронирования

### Bookings (`/admin/bookings`)

Аккордеон со всеми событиями. При раскрытии события подгружаются его бронирования. Для каждого бронирования доступны удаление и отметка оплаты.

### WidgetSettings (`/admin/widget`)

Визуальный редактор виджета: основной цвет, фон, шрифт, радиус скруглений, текст кнопки, сообщение после бронирования, произвольный CSS. Предпросмотр — живая iframe с виджетом.

### AppSettings (`/admin/settings`)

Настройки организации (название, контакты) и SMTP для отправки документов. Изменения применяются без перезапуска сервера.

## Типы (`types/index.ts`)

| Интерфейс | Описание |
|-----------|----------|
| `Event` | Событие (название, даты, цена, вместимость, статус, ценовые варианты) |
| `EventFormData` | Форма создания/редактирования события |
| `Booking` | Бронирование (гостевые данные, оплата, комментарий, документы) |
| `AdminBookingRequest` | Запрос на создание бронирования администратором |
| `BookingSummary` | Счётчики мест и бронирований для события |
| `DocumentTemplate` | Шаблон документа |
| `BookingDocument` | Документ, привязанный к бронированию |
| `WidgetConfig` / `WidgetConfigUpdateRequest` | Конфигурация виджета |
| `AppSettings` | Настройки приложения |

## Docker

```bash
docker build -t soldo-admin .
docker run -p 3000:80 soldo-admin
```

`Dockerfile` — multi-stage: `node:20` (сборка Vite) → `nginx:1.27` (раздача статики). Nginx настроен с SPA fallback (`try_files $uri /index.html`).
